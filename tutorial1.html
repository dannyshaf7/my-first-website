Exploiting Web Authentication Vulnerabilities

  1. When looking for web app vulnerabilities, we often start without credentials.
  1. Bypassing the login altogether, when possible, is a great start.
  2. The robots.txt file can sometimes expose sensitive information.
  3. It is used by website owners to communicate with search engines
    and tell them which parts not to crawl. Useful information can
    sometimes be found here, including sensitive or hidden directories and files.
  4. Parse these directories, test pages and old website versions for sensitive data.
  5. In this vulnerable webserver we find a hidden directory for an admin portal.
  1. Navigating to the admin portal in this way bypasses the login entirely
    and gives us access to the administrator action of deleting users!
  1. Sensitive information can also be found in the network traffic sent to and from the webserver.
  2. Burp Suite is a useful tool for intercepting network traffic to better understand a site.
  3. Here, we open a Chromium browser hosted by the Burp Suite session and navigate to our target.
  1. Attempt to log in to the My account portal on the website.
  2. Return to the Burp Suite Target window and view the /login response.
  1. Look through the Response script for sensitive information left in by the web developers.
  2. Here we can see an admin panel tag attribut in plain text that appears to be a directory
  3. Next we should navigate to the address to see where it takes us.
  1. Navigating to the directory we found in the login response takes us to the admin panel,
    bypassing the login entirely and giving us access to the administrator action of deleting users!
  1. We can also view network traffic without installing tools like Burp Suite.
  2. Browsers provide powerful tools, often called Web Developer Tools, within the browser itself.
  3. Here, we navigate to the menu, more tools, where we find the Web Developer Tools option.
  1. Follow the same steps to attempt login.
  2. Navigate to the Network tab of the Web Developer Tools menu and find the login packet response.
  3. Here you will see the same script we reviewed in Burp Suite which shows the admin panel directory in plain text.
  4. Note: the directory name is different each time the lab is opened.
  1. In this scenario, we have been given low-level user credentials without administrator rights.
  2. Even though we can log in, we need to find a way to elevate our priveleges.
  1. Another powerful tool in web development and operation is the cookie.
  2. A cookie is a small piece of data web browsers use to store and retrieve information.
  3. They can provide a faster and more personalized experience for the user.
  4. With this conveniece comes risk. Cookies can be used to keep users logged in, remember login
    credentials, track user behavior across websites and more. This data should be carefully guarded.
  5. We start by opening the website in Burp Suite browser.
  1. This time we navigate to the Proxy tab of Burp Suite to enable packet interception.
  2. With the proxy, we are able to review and make changes to packets before they are sent to the web.
  1. Enter the user login credentials and click log in. You will notice nothing happens in the browser.
  2. This is because the Burp Suite proxy is intercepting the action. Return to Burp Suite.
  1. Burp Suite displays the details of the request packet and allows changes before forwarding to the web.
  2. Notice that there is a session cookie attribute of Admin which is set to false.
  3. Here we can find vulnerabilities in how cookies are handled that may allow us to manually update the value.
  1. Change the value of Admin to "true" from "false".
  2. Forward the request packet to the web to log in using the credentials and cookie value of Admin=true.
  3. This may require clicking Forward multiple times depending on how many packets are needed
  4. Return to the Burp Suite Chromium browser window.
  1. Now that we are logged in, we can navigate to /admin address to access the admin panel.
  2. Changing the cookie has granted access to the administrator panel and the action of deleting users!
